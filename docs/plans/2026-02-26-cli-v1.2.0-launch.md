### Task 2: Fix Windows path issue in open.ts

`getCurrentProjectName()` uses hardcoded `/` to split paths — fails on Windows.

**Files:**
- Modify: `cli/src/commands/open.ts:59-66`

**Step 1: Replace hardcoded split with path.basename**

Ensure `import path from 'path';` exists at top of file. Then change:
```typescript
function getCurrentProjectName(): string | null {
  try {
    const cwd = process.cwd();
    const parts = cwd.split('/');
    return parts[parts.length - 1] || null;
  } catch {
    return null;
  }
}
```
to:
```typescript
function getCurrentProjectName(): string | null {
  try {
    return path.basename(process.cwd()) || null;
  } catch {
    return null;
  }
}
```

**Step 2: Verify build**

```bash

```
Expected: Clean compile, no errors.

**Step 3: Commit**


### Task 3: Fix Windows path issue in jsonl.ts extractProjectName

`extractProjectName()` at line 355-358 splits on hardcoded `/`. The input comes from `extractProjectPath()` which always returns forward-slash paths (by design — it's decoding Claude's encoded project directory). But `path.basename()` is safer and cleaner.

**Files:**
- Modify: `cli/src/parser/jsonl.ts:355-358`

**Step 1: Replace split with path.basename**

Note: `extractProjectPath()` returns decoded paths like `/Users/name/project`. These always use `/` regardless of platform (they're decoded from Claude's encoding, not from OS paths). However, `path.basename()` handles both separators on Windows, so it's the safer choice.

Change:
```typescript
function extractProjectName(projectPath: string): string {
  const parts = projectPath.split('/').filter(Boolean);
  return parts[parts.length - 1] || 'unknown';
}
```
to:
```typescript
function extractProjectName(projectPath: string): string {
  // projectPath comes from Claude's encoded format (always forward slashes)
  // path.basename handles both separators cross-platform
  return path.basename(projectPath) || 'unknown';
}
```

**Step 2: Verify build**

```bash
cd cli && pnpm build
```
Expected: Clean compile.

**Step 3: Commit**

```bash
git add cli/src/parser/jsonl.ts
git commit -m "fix(cli): use path.basename in extractProjectName for Windows compat"
```
